Parser File Syntax

The notation used for representing the parser file syntax in this document is described here.

Keywords

parser‑keyword → cppkeyword | api | empty | include | main | parser | ruleinfo | uselexer | var
parser‑file → parser‑file‑declaration*
parser‑file‑declaration → include‑declaration | using‑namespace‑declaration | parser‑declaration
include‑declaration → include‑prefix? # include file‑path
include‑prefix → cpp‑prefix | hpp‑prefix
cpp‑prefix → [cpp]
hpp‑prefix → [hpp]
using‑namespace‑declaration → using namespace qualified‑cpp‑id? ;
parser‑declaration → parser api? identifier { parser‑statement* }
parser‑statement → main‑statement | using‑statement | use‑lexer‑statement | rule‑info‑statement | rule‑statement
main‑statement → main ;
using‑statement → using parser‑rule‑id ;
parser‑rule‑id → identifier (. identifier)*
use‑lexer‑statement → uselexer identifier ;
rule‑info‑statement → ruleinfo { rule‑info* }
rule‑info → ( identifier , string‑literal )
rule‑statement → rule‑name params‑and‑vars? return‑type? ::= rule‑body ;
rule‑name → identifier
params‑and‑vars → ( param‑or‑var (, param‑or‑var)? )
param‑or‑var → var type‑id declarator | type‑id declarator
return‑type → : type‑id
rule‑body → alternative
alternative → sequence (| sequence)*
sequence → difference difference*
difference → list (- list)*
list → postfix (% postfix)?
postfix → primary (* | + | ?)?
primary → ( rule‑call | primitive | grouping ) expectation? action?
rule‑call → nonterminal ( ( expression‑list ) )? : identifier
nonterminal → identifier
primitive → empty | identifier
grouping → ( alternative )
expectation → !
action → compound‑statement (/ compound‑statement)?
