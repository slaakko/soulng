Installation

Installation on Windows

Installation using a setup program

  • Download the setup executable and run it.
  • The SoulNG is by default installed to the C:\soulng directory.
  • The C:\soulng\bin directory is added to the system PATH environment variable.
  • The setup creates a system environment variable SOULNG_ROOT that contains the path to the installation directory.
  • The SOULNG_ROOT environment variable is needed at run-time to locate the Unicode character database soulng_ucd.bin in the unicode subdirectory.

Installation from sources

Requires Boost C++ libraries. The exact version does not matter as far it is a reasonably recent version. The soulng binaries have been built using Boost version is 1.77.

Download

  • Download a Soulng 3.1.0 Source Package
  • Extract it to a directory of your choice here called C:\soulng.

Configuration

  • Open C:\soulng\config\build.props file with a text editor.
  • Set the contents of the BOOST_INCLUDE_DIR element to the path to your Boost include directory. The default value is C:\boost\include\boost-1_77.
  • Set the contents of the BOOST_LIB_DIR element to the path to your Boost library directory. The default value is C:\boost\lib.

Compilation

  • Make an environment variable SOULNG_ROOT with value C:\soulng.
  • Add the C:\soulng\bin directory to the PATH environment variable.
  • Open x64 Native Tools Command Prompt for VS 2019
  • Change to C:\soulng\nmake directory.
  • Run nmake.

Installation on Linux / on WSL

Building and installing the binaries

  • Download the SoulNG 3.0.0 Source Package.
  • Extract it to a directory of your choice here called soulng-3.0.0
  • change to the soulng subdirectory: 
                    slaakko@pluto:~/soulng-3.0.0$ cd soulng
  • run make: 
                    slaakko@pluto:~/soulng-3.0.0/soulng$ make
  • Set an environment variable with name SOULNG_ROOT that points to the soung-3.0.0 directory. 
                    slaakko@pluto:~/soulng-3.0.0/soulng$ export SOULNG_ROOT=~/soulng-3.0.0
    You have to put it to your profile script to make it permanent. The SOULNG_ROOT environment variable is needed at run-time to locate the Unicode database that is in the soung-3.0.0/unicode subdirectory.
  • run [sudo] make install 
                    slaakko@pluto:~/soulng-3.0.0/soulng$ sudo make install
    The binaries are installed by default to the /usr/local/bin, libraries to the /usr/local/lib, and headers to the /usr/local/include/soulng directory. You can set the installation location by giving the prefix argument:
    [sudo] make prefix=/where/to/install install.

Building and installing the examples

  • change to the examples subdirectory: 
                    slaakko@pluto:~/soulng-3.0.0$ cd examples
  • run make: 
                    slaakko@pluto:~/soulng-3.0.0/examples$ make
  • run [sudo] make install: 
                    slaakko@pluto:~/soulng-3.0.0/examples$ sudo make install
    The example binaries minilang and cmajortester are installed by default to the /usr/local/bin directory. You can set the installation location by giving the prefix argument:
    [sudo] make prefix=/where/to/install install.

Building and installing XML processing libraries (optional)

  • change to the sngxml subdirectory: 
                    slaakko@pluto:~/soulng-3.0.0$ cd sngxml
  • run make: 
                    slaakko@pluto:~/soulng-3.0.0/sngxml$ make
  • run [sudo] make install: 
                    slaakko@pluto:~/soulng-3.0.0/sngxml$ sudo make install

Building and installing C++ front-end libraries (optional)

  • change to the sngcpp subdirectory: 
                    slaakko@pluto:~/soulng-3.0.0$ cd sngcpp
  • run make: 
                    slaakko@pluto:~/soulng-3.0.0/sngcpp$ make
  • run [sudo] make install: 
                    slaakko@pluto:~/soulng-3.0.0/sngcpp$ sudo make install
