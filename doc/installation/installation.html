Installation

Installation on Windows

  • Download the setup executable and run it.
  • The SoulNG is by default installed to the C:\soulng-2.0.0 directory.
  • The C:\soulng-2.0.0\bin directory is added to the system PATH environment variable.
  • The setup creates a system environment variable SOULNG_ROOT that contains the path to the installation directory, that is by default C:\soulng-2.0.0.
  • The SOULNG_ROOT environment variable is needed at run-time to locate the Unicode character database soulng_ucd.bin in the unicode subdirectory.

Installation on Linux / on WSL

Building and installing the binaries

  • Download the SoulNG 2.0.0 Source Package.
  • Extract it to a directory of your choice here called soulng-2.0.0
  • change to the soulng subdirectory: 
                    slaakko@pluto:~/soulng-2.0.0$ cd soulng
  • run make: 
                    slaakko@pluto:~/soulng-2.0.0/soulng$ make
  • Set an environment variable with name SOULNG_ROOT that points to the soung-2.0.0 directory. 
                    slaakko@pluto:~/soulng-2.0.0/soulng$ export SOULNG_ROOT=~/soulng-2.0.0
    You have to put it to your profile script to make it permanent. The SOULNG_ROOT environment variable is needed at run-time to locate the Unicode database that is in the soung-2.0.0/unicode subdirectory.
  • run [sudo] make install 
                    slaakko@pluto:~/soulng-2.0.0/soulng$ sudo make install
    The binaries are installed by default to the /usr/local/bin, libraries to the /usr/local/lib, and headers to the /usr/local/include/soulng directory. You can set the installation location by giving the prefix argument:
    [sudo] make prefix=/where/to/install install.

Building and installing the examples

  • change to the examples subdirectory: 
                    slaakko@pluto:~/soulng-2.0.0$ cd examples
  • run make: 
                    slaakko@pluto:~/soulng-2.0.0/examples$ make
  • run [sudo] make install: 
                    slaakko@pluto:~/soulng-2.0.0/examples$ sudo make install
    The example binaries minilang and cmajortester are installed by default to the /usr/local/bin directory. You can set the installation location by giving the prefix argument:
    [sudo] make prefix=/where/to/install install.

Building and installing XML processing libraries (optional)

  • change to the sngxml subdirectory: 
                    slaakko@pluto:~/soulng-2.0.0$ cd sngxml
  • run make: 
                    slaakko@pluto:~/soulng-2.0.0/sngxml$ make
  • run [sudo] make install: 
                    slaakko@pluto:~/soulng-2.0.0/sngxml$ sudo make install
